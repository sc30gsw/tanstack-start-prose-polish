settings.json の hooks で定義された自動化です。Edit|Write 後に走るチェックと、Stop 時の継続診断があります。

イベントマッチャ実行動作
PostToolUseEdit|Write—編集または作成されたファイルが .ts/.tsx/.js/.jsx/.json/.css なら、そのファイルが属する Git プロジェクトルートに移動して `vp fmt <file>` を実行し、対象ファイルを自動フォーマットする。
PostToolUseEdit|Write—編集または作成されたファイルが .ts/.tsx なら、そのプロジェクトルートで `vp check` を実行し、フォーマット・Lint・型チェックを行い、失敗時は stderr に警告メッセージを出す（exit 0 で常に正常終了）。
PostToolUseEdit|Write—対象ファイル内に相対パスインポート（`from './' や '../'`）が含まれていないか grep で検査し、見つかった場合は該当行を stderr に表示して `~/` エイリアスを使うよう警告する。
PostToolUseEdit|Write—対象ファイル内に `interface` 宣言が含まれていないか grep で検査し、見つかった場合は該当行を stderr に表示して `type` を使うよう警告する。
PostToolUseEdit|Write—対象ファイルが /routes/ ディレクトリ外にある場合に `export default` が含まれていないか grep で検査し、見つかった場合は該当行を stderr に表示して named export を使うよう警告する（default export は src/routes/ のみ許可）。
PostToolUseEdit|Write—対象ファイル（テストファイルを除く）に try/catch 構文が含まれていないか grep で検査し、見つかった場合は該当行を stderr に表示して better-result パターン（Result.tryPromise / result.match）を使うよう警告する。
Stop*asyncターンの終了時に `${CLAUDE_PROJECT_DIR}/.claude/hooks/claude-doctor-continuous.sh` をバックグラウンドで実行し、claude-doctor によるアンチパターン検出とルール候補レポートの生成を行う。

フックスクリプト

claude-doctor-continuous.sh

.claude/hooks/claude-doctor-continuous.sh

Stop フックとして各ターン終了時に呼び出され、10分間のスロットリングを挟みつつ claude-doctor をバックグラウンドで実行し、アンチパターン検出結果とルール候補をタイムスタンプ付きの JSON/ログとして .claude/claude-doctor-reports/ に蓄積する。perl の alarm を使った macOS 互換のタイムアウト(120秒)で claude-doctor を起動し、Claude Code の UX を妨げないよう完全にデタッチして実行する。

スクリプト全文を表示
#!/bin/bash
# claude-doctor-continuous.sh
#
# Stop hook: At end of each turn, run claude-doctor to detect anti-patterns and emit rule candidates.
# Throttling (10 minutes) and background execution avoid impacting Claude Code UX.
#
# Operations workflow:
#   1. While you work, the hook quietly accumulates reports in the background
#   2. Periodically inspect the latest report: ls -lt .claude/claude-doctor-reports/ | head
#   3. Pick rules to adopt and append them to ~/.claude/CLAUDE.md or each project's CLAUDE.md
#   4. Remove old reports: find .claude/claude-doctor-reports -mtime +30 -delete

# Discard stdin so the hook runtime does not wait for the pipe to close
cat > /dev/null

HOOK_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
PROJECT_ROOT="${CLAUDE_PROJECT_DIR:-$(cd "$HOOK_DIR/../.." && pwd)}"

LAST_RUN_FILE="$PROJECT_ROOT/.claude/.cache/claude-doctor-last-run"
REPORTS_DIR="$PROJECT_ROOT/.claude/claude-doctor-reports"

# Ensure directories exist
mkdir -p "$(dirname "$LAST_RUN_FILE")" "$REPORTS_DIR" || true

# Skip if we already ran within the last 10 minutes
if [ -f "$LAST_RUN_FILE" ] && find "$LAST_RUN_FILE" -mmin -10 2>/dev/null | grep -q .; then
  exit 0
fi

# Record run time for the next throttle check
touch "$LAST_RUN_FILE" || true

# Output paths with timestamp
TIMESTAMP=$(date +%Y%m%d-%H%M%S)
REPORT_JSON="$REPORTS_DIR/${TIMESTAMP}.json"
REPORT_LOG="$REPORTS_DIR/${TIMESTAMP}.log"

# Run claude-doctor in the background (close stdin for full detachment)
# Use perl alarm as a macOS-compatible timeout (no GNU coreutils required)
{
  cd "$PROJECT_ROOT" || exit 0
  perl -e 'alarm 120; exec @ARGV' -- \
    npx -y claude-doctor@latest --project "$PROJECT_ROOT" --rules --save --json \
    > "$REPORT_JSON" \
    2> "$REPORT_LOG" \
    || true
} </dev/null &
disown

exit 0
